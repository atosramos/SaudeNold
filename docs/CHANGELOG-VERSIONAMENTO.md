## Changelog e versionamento

### Padrao de versionamento
Usamos **SemVer**: `MAJOR.MINOR.PATCH`
- **MAJOR**: mudancas que quebram compatibilidade
- **MINOR**: novas funcionalidades compat√≠veis
- **PATCH**: correcoes e melhorias pequenas

### Como registrar mudancas
Registre mudancas relevantes em um changelog por release.
Sugestao de secoes:
- Added
- Changed
- Fixed
- Security

### Exemplo de entrada
## 1.2.0
- Added: notificacao de novos logins
- Fixed: refresh token revogado ao bloquear dispositivo

### Quando atualizar a versao
- **MAJOR**: alteracao de contrato de API, quebra de fluxo principal
- **MINOR**: novas features relevantes
- **PATCH**: correcoes, ajustes visuais, refactors
